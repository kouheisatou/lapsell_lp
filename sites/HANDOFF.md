## 1. プロジェクト概要

**Lapsell** は、アーティストが「練習の時間枠」を先に出品し、当日スマホで撮影した映像を約10秒のセグメントに分割して「世界に1つの動く一点物グッズ」としてランダム配布するマーケットです。

このプロジェクトでは、その **ランディングページ (LP)** を **13案の全く異なる世界観で並列制作**しています。最終的にユーザー（オーナー）が13案から1つを選びます。

- **LPの読者 = 出品者**（ライブアイドル / インディーズバンド・楽曲アーティスト / インディーズお笑い芸人）
- **LPタイトル**: 「練習時間を、収益に変える。」
- **ゴール**: 出品者が「使えば自分の練習時間が売れる / 課題が解決する」と**具体的にイメージできる**こと。

## 3. 制作の制約（全案共通）

各LPは **単一HTML** で成立させます。

- CSS / JS はインライン。**外部CDN・外部フォント非依存**。
- 画像は**相対パス**。全 `<img>` に `alt` と `loading="lazy"` を付与。
- **スマホ375px幅で横スクロールが発生しないこと**。

### 禁止事項
1. 枠カード地獄（カードだらけのレイアウト）
2. 青紫 / 青系グラデの角丸ボタン
3. 数字を大きくドン出しした後に説明する構成
4. ふわふわ曖昧 / ポエム表現
5. AIっぽい万能イラスト

**分かりやすさ最優先。**

### 制作方式
各サイトは `sites/BASE_PROMPT.md` ＋ **自分の `concept.md` だけ**を見て独立設計します（他案は見ない）。

## 5. 重大な制約: 画像生成の spend cap

codexの画像生成が**ワークスペースの spend cap 上限に到達**し、写真の追加生成は**当面ほぼ不可**です（オーナー＝ユーザーが上限を上げたが、多数のエージェント一斉生成で即枯渇）。

- 画像が少ないサイト（02・03・10・18・05 等）は**既存写真＋CSS/HTMLで成立**させています。
- cap回復後に写真追加の余地あり。

## 7. 運用知見（ハマりどころ）

### ローカルHTTPサーバ
- PowerShell の `Start-Process` でデタッチ起動すること。Bashの `run_in_background` や `nohup` は別ツール呼び出しから到達できない / すぐ死ぬ。
- 起動例:
  ```
  Start-Process -FilePath "python" -ArgumentList "-m","http.server","8137","--bind","127.0.0.1" -WorkingDirectory "C:\Users\usr0107095\Desktop\新しいフォルダー\sites" -WindowStyle Hidden
  ```
- **現在ポート8137で稼働中**。各LPは `http://127.0.0.1:8137/<案フォルダ名>/` で閲覧。

### file:// が使えない
- 日本語パス（"新しいフォルダー"）だとChromeが弾く。**必ずHTTPサーバ経由**で開く。

### codex画像生成
- 日本語パスへの直接保存はOK。`C:/temp` 等はsandbox外で書き込み拒否。
- コマンド: `codex exec --sandbox workspace-write --skip-git-repo-check '$imagegen <プロンプト>' < /dev/null`
  - 末尾に**保存先の絶対パスを明示**。Bashタイムアウトは 300000ms。
- ただし現在は spend cap 到達のため生成はほぼ不可（5章参照）。

### ブラウザ確認
- 共有の Claude-in-Chrome MCP は複数エージェントで競合しタブが乱れる。
- **各エージェントは自前の隔離ヘッドレスChrome**（専用 `--user-data-dir`・別デバッグポート）で確認する。
