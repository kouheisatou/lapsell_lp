# Lapsell LP 制作ルール（ユーザー指摘の全反映チェックリスト）

> 本ファイルは、これまでにユーザーから受けた**すべての指摘**をルール化したもの。全15LPと今後のfan-outは、ここ＋`lapsell_brief.md` を**必ず満たす**こと。各LP完成時に下のチェックを全項目通す。

---

## A. コンテンツ・正確性
- [ ] **コアバリューは `サービス設計/Lapsell.pdf`（全23p）＋`Lapsell_サービス設計書.md`＋`lapsell_lp_設計.md` を読み直して理解**し、LPに落とし込む。`lp_構成md`の文言・構成に引っ張られすぎない。参考LPの世界観に合わせて柔軟に。
- [ ] **数字はピッチ実値が正**。特に **推し活市場(TAM)=約4,650億円/年**。「**4.65兆円」は誤りで使用禁止**。
- [ ] 実績数字は実スクショの値：BLACKPINK練習 **20億回視聴**／M!LK「イイじゃん」**4,175万回視聴・第76回NHK紅白歌合戦歌唱曲**／SEVENTEEN「練習風景だけで笑える10年目アイドル」**895,794回再生・高評価2万超**。
- [ ] SEOで自然流入→申し込みに繋がる構成（title/meta/H1にKW、`Service`＋`FAQPage` JSON-LD）。
- [ ] スタンス：**プロのLPデザイナー兼スタートアップコンサルとして「最高にわかりやすい」LP**。

## B. コピー・言い回し（AI臭の排除）
- [ ] **抽象コピー禁止**：「『コピーされたら？』を、技術で消す。」のような、画像でイメージできない決め台詞は使わない。常に画面・図・実例で具体化。
- [ ] **「さあ、何かな…?」のようなAI臭い決め台詞は禁止**（開封画面はピッチ実物の「あなたの区間が届きました／タップして開ける」を使う）。
- [ ] **「順番が逆。」は使わない**（仕組み見出しから削除）。
- [ ] **キャッチーさより「わかりやすさ」優先**。
- [ ] **「地下アイドル」は使わない（お客様＝出品者に失礼）。**「**ライブアイドル**」（または「インディーズアイドル」）等、敬意ある呼称を使う。SEOキーワードも同様。
- [ ] **他者・他作品を下げる断定表現は使わない**（例「**一番**価値ある“裏側”」＝完成品より上、と言い切るのはクリエイターに失礼）。「価値ある“裏側”」「見たいのに届かない“裏側”」のように、誰も否定しない言い方に。

## C. 数字・データの見せ方
- [ ] **「数字をドンと大きく出して後から説明」はAI臭の典型＝禁止**。数字は必ず**わかる一文の中**か**グラフ＋説明見出し**で示し、ぱっと見で「何の数字か・なぜ重要か」が伝わるように。
- [ ] 実績カードも「20億回」を大きく出すだけにせず、**結論を見出し**に（例「『練習動画』が公式MVより観られている。」）、数字は文中で補強。
- [ ] **エビデンス・数値はグラフ/チャートで可視化**（HTML/CSS/inline SVG、ライブラリ不可）。例：支払い意向ドーナツ（約3人に1人＝31.9%）／ガチャ7,000億・トレカ2,349億・ライブBD558億の比較バー／75.3%ドーナツ。
- [ ] **TAM/SAM/SOM・事業計画（3年GMV等）の投資家向けチャートは不要**＝載せない（販売者向けLPのため）。

## D. デザイン
- [ ] **アイコンは線のシンプルなアイコンのみ**（inline SVG, stroke 1.5–2px, currentColor）。**絵文字禁止**。
- [ ] **「枠で囲みすぎる」AI臭を排除**。余白・タイポ階層・整列・背景帯で構造を作る。罫線/カードは参考サイトが実際に使う所だけ。影・角丸・グラデ枠の乱用禁止。
- [ ] 各LPは担当参考サイト（`references/NN_name/analysis.md`＋スクショ）の世界観に**完全に合わせて独自に**作る。**共通CSS/JSは作らない**。
- [ ] レスポンシブ必須・ページ全体の横スクロール禁止（1440 / 390）。※利用フローのステップ列など、意図した横スクロール領域は可（G参照）。
- [ ] **黒（暗）背景に黒文字を置かない**。暗背景セクションのテキストは白／近白に。**黄色マーカー（ハイライト）は、暗背景では「透明下線」ではなく「塗りつぶし黄＋黒文字」にして必ず可読に**（透明下線＋黒文字だと暗背景で文字が消える＝過去の不具合）。ヒーローの見出し強調語も同様。

## E. 画像・図（作り分け）
- [ ] **各アピールポイントにイメージしやすい画像**（codex生成イラスト）を添える。
- [ ] **課題セクションにも画像**を添える（ファン側／アーティスト側）。
- [ ] **ヒーローはダサいAI人物画像をやめ、「練習動画が販売されているメルカリのような画面」のスマホアプリモック**を、いい感じにヒーローに配置する。
- [ ] **他社比較は「表」で**見せる（2×2マップではなく比較テーブル）。
- [ ] 画像はcodex（`/codex-imagegen` ＝ ChatGPT image / gpt-image）で生成。**それぞれのLPに合った画像を個別生成し、LP間で使い回さない**。
- [ ] **実際にバズった実績はclaude in chromeで実スクショを取得**し、プロジェクト内に配置して適切な位置に埋め込む（実画像は外部ファイルOK）。
- [ ] **主要セクションにも納得感のある画像を**：アピールだけでなく **課題 / 利用フロー / For Artists（対象）/ Comparison（他社比較）/ Pricing（料金）/ Authenticity（真贋・所有）** にも、イメージできる画像をcodexで生成して「いい感じに」配置する。
- [ ] **重要ビジュアルは改善ループを回す**：課題画像など意図が伝わるべき画像は「生成 → 生成画像をReadで読み取り → 課題/意図を的確に表せているか検証 → プロンプト改善して再生成」を、的確になるまで繰り返す（一発で決めない）。
- [ ] **画像の比率・サイズを壊さない**：横長スクショ（YouTube≒4.5:1）は `width:100%;height:auto` で全体表示（cover切り抜き禁止＝見出し/数字が欠ける）。縦長と横長を同じグリッドに混ぜない（縦長は専用枠）。アプリ縦画面を全幅で巨大化させない。

## F. アプリモック（利用フロー）
- [ ] **アプリ画面は静的HTMLコンポーネントで自作**し、**スマホ風の黒枠**をあてがって「いい感じ」に。フレーム・UI・文字・ボタンはHTML（鮮明）。
- [ ] **モック内の写真・イラスト部分（練習動画の静止画・カプセル等）はcodex生成画像を`<img>`で埋め込む**。
- [ ] **利用フローは分割せず、単一の「5ステップ」**：①出品 → ②一覧から購入（メルカリ風の一覧で時間枠が並ぶ）→ ③撮影（スマホで撮るだけ／RECモック）→ ④届く・開封（「あなたの区間が届きました／タップして開ける」）→ ⑤受け取り（世界に1つを所有）。
- [ ] **ステップは「横スクロール」で見せる**（5枚を横並び＋overflow-x。ページ全体の横スクロールは出さない）。
- [ ] **各ステップに「販売者」／「購入者」タグ**を付け、どちら側の操作か一目でわかるように（①③＝販売者、②④⑤＝購入者）。
- [ ] **メルカリ風の一覧（時間枠が並ぶ）**と**購入の動き**を必ず含める。
- [ ] **ヒーローのスマホモックは「3Dで浮遊」させると良い**（世界観が合うLP＝05/08/09/14等）：わずかに傾け（例 `transform: perspective(1600px) rotateY(-10deg) rotateX(4deg)`）、**真下に楕円のソフトな影**（ぼかした影 or `::after`の楕円）を敷いて浮いて見せる。やりすぎず上品に。
- [ ] **モック内の文字は小さく洗練**（実アプリの比率。大きすぎ＝ダサい）。モックは**等幅・同サイズ・黒枠統一**、中身は世界観配色。

## G. 構成・セクション（推奨順／参考サイトに合わせ可変）
> **【最重要・最優先ルール】各LPの「セクション構成・順序・コンテンツ種類・ページの語り口」は、担当する参考サイト自身のページ構造を踏襲する。** 下記のLapsell要素リストは“固定のレイアウト/順序”ではなく**「盛り込むべき内容のチェックリスト」**にすぎない。**全LPに同じスケルトンを当てはめるのは禁止**（＝今までの失敗。色やブロックを変えただけになる）。参考サイトごとに section の数・順序・名前・見せ方が**大きく異なってよい**し、異ならせること。固定構成に引っ張られず、自由に。
> 手順：①参考サイトの screenshots + analysis.md を見て、その**実際のセクション構成（何が・どの順で・どんな見せ方か）を書き出す** → ②Lapsellの内容を**その構造に流し込んで**再構築する。例：Pockyならキャンペーン体験サイトの構成（KV→テーマ選択→体験→ギフト/シェア）、Felissimoならクラファン読み物の構成（物語→共感→支援/達成）、iGrowなら金融アプリの簡潔な構成、というように**まるごと別構造**にする。

盛り込む内容（チェックリスト＝順序自由、参考サイトの構造に合わせて配置・取捨）：Lapsellアプリの黒枠HTMLモック（どこかに）／コア機構（練習の“時間”を先に売る→ガチャ→世界に1つ）／課題（ファン＆作り手）／4つの価値／実績（チャート＋実スクショ）／真贋・所有／他社比較／料金（0円・5%〜）／対象アーティスト／FAQ（JSON-LD）／CTA。
（※TAM/SAM/SOM・事業計画は入れない。各所に納得感ある画像を添える）

## H. 技術・運用
- [ ] **単一HTMLファイル**：CSS/JSはHTMLに埋め込み。**画像だけ外部ファイル**（各LPの `assets/`）。
- [ ] **各LPはフォルダを切り**、生成画像をその `assets/`（実績は `assets/evidence/`）に配置。
- [ ] **git管理は親リポジトリ1つだけ**。サブフォルダに `.git` を作らない（**codexは必ずリポジトリ直下をcwdで実行**）。LP内でgitコマンドを実行しない。
- [ ] **簡易サーバーを立て→Chromeで開き→claude in chromeで実機レビュー→自己フィードバックを回して修正**（ブラウザキャッシュに注意。確認時はハードリロード）。
- [ ] パイロット（01_givee）で品質基準を確定→OK後に残りを並列展開。
- [ ] **gitの旧実装は参照しない**（意図的に削除済み）。
